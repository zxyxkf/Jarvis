# Jarvis

Jarvis 是一个面向电商团队的企业 AI 工作助手，目标是把分散的商品资料、图片素材、客服话术、知识文档和团队对话统一沉淀到公司自己的知识库中，让团队成员可以在网页端、桌面端以及未来的移动端/小程序端同步维护和使用。

项目当前重点不再是通用文档知识库，而是逐步收敛为“电商商品资料库 + 团队 AI 助手”。团队成员可以围绕某个供货商、品牌、系列或业务场景建立知识库，在其中维护商品、款式、SKU、素材、FAQ、导入记录和操作记录。后续助手会基于这些数据检索商品信息、图片和客服话术，帮助客服、运营、选品、售后等角色更快、更准确地完成日常工作。

## 产品定位

Jarvis 希望解决电商团队中常见的资料查找和维护问题：

- 商品款式、SKU、价格、库存、图片和卖点分散在不同文件、群聊或平台后台中，查找效率低。
- 客服沟通时需要快速确认商品图片、规格、注意事项和标准话术，人工翻找容易慢或出错。
- 多人维护资料时容易重复导入、重复新增、覆盖人工维护数据。
- Web、桌面端、未来移动端/小程序端需要访问同一份数据和同一套对话记录。
- 后期需要预留电商平台、ERP 或其他系统的开放 API 同步能力。

Jarvis 的核心方向是让商品资料成为团队可维护、可检索、可追溯、可同步的数据资产，而不是散落在表格和聊天记录里的临时信息。

## 当前建设方向

### 商品资料库

商品资料库以知识库为单位组织数据。一个知识库可以理解为一个供货商、一个品牌、一个项目或一个业务资料集合。

当前设计重点包括：

- 商品主信息：商品名、分类、品牌、款式、卖点、详情、注意事项、状态、来源等。
- 款式与 SKU：一个商品可以有多个款式，一个款式下可以维护多个 SKU。款式是商品资料去重和导入合并的重要依据。
- 素材库：图片、视频、文件统一进入素材库，再关联到商品，支持主图和多图。
- 商品 FAQ 和客服话术：围绕商品维护常见问题、回复口径和注意事项。
- 表格导入：支持 Excel/CSV 导入商品和 SKU，并对重复数据进行识别、跳过或补全。
- 操作记录：记录关键维护行为，便于追踪谁在什么时间操作了哪些商品资料。

### 团队 AI 助手

Jarvis 的助手能力会逐步从“回答文档问题”升级为“理解商品资料并辅助电商工作”：

- 根据客服或运营的自然语言问题检索商品、款式、SKU、图片和话术。
- 返回结构化商品卡片，而不是只返回一段纯文本。
- 支持按知识库选择检索范围，例如只查某个供货商或某个品牌资料。
- 对商品卖点、注意事项、售后说明进行整理和生成。
- 结合团队历史对话和知识库内容，形成可持续沉淀的工作助手。

### 三端同步

Jarvis 的数据以服务端为中心，Web、桌面端和未来移动端/小程序端都访问同一套 API。

同步原则：

- 商品资料、素材、知识库、导入记录、操作记录和对话记录都存储在服务端。
- Web 端和桌面端复用同一套核心页面和业务接口。
- 移动端/小程序端后续不单独设计一套割裂的数据结构，而是基于现有 API 扩展。
- 团队成员在任意端新增、编辑、导入或查询，其他端都能看到一致的数据。

## 未来发展方向

### 1. 商品资料库完善

- 完善商品详情编辑体验，覆盖基础信息、商品属性、销售属性、详情内容、素材和话术。
- 强化重复数据识别，避免不同成员重复新增同一商品或同一款式。
- 导入时支持更灵活的补全策略：已有商品和款式不重复创建，只新增缺失 SKU 或补全空字段。
- 支持更完整的导入记录、错误行提示和可追溯操作日志。

### 2. 助手接入商品检索

- 聊天时同时检索文档知识和商品资料。
- 命中商品后返回商品名、款式、SKU、图片、价格、库存状态、卖点和话术。
- 前端消息区展示商品卡片，方便客服直接复制信息或查看图片。
- 后续支持更智能的商品推荐、相似款查找和多商品对比。

### 3. 平台 API 同步

后期会预留并逐步接入淘宝、抖店、拼多多、ERP 或其他电商平台开放接口。

同步策略默认保持保守：

- 只补充本地资料库中不存在的商品、款式或 SKU。
- 默认不覆盖人工维护或表格导入的数据。
- 平台数据与本地数据通过映射表记录来源和外部 ID。
- 每次同步前提供预览，让团队确认将新增、跳过或冲突的数据。

### 4. 权限与协作

第一阶段以团队共享和全员可维护为主，后续会逐步补充更细的协作能力：

- 按知识库、角色或成员控制访问与编辑权限。
- 关键资料修改支持操作记录和版本追踪。
- 对导入、删除、批量修改等高风险操作增加更清晰的确认和恢复能力。
- 面向客服、运营、管理员提供不同的工作视图。

### 5. 移动端和小程序

移动端/小程序会作为后续阶段推进，重点用于轻量查询和快速维护：

- 手机端快速搜索商品、SKU、图片和话术。
- 外出或仓库场景下快速查看商品信息。
- 支持拍照上传素材或补充商品资料。
- 与 Web、桌面端保持同账号、同知识库、同对话记录同步。

## 技术概览

当前项目采用 monorepo 结构：

- `apps/web`：Web 前端，基于 Vue、Vite、Pinia。
- `apps/desktop`：桌面端，基于 Tauri 和 Vue。
- `apps/server`：服务端，基于 NestJS、Prisma。
- `packages/shared`：前后端共享类型。

基础服务方向：

- PostgreSQL / pgvector：业务数据和后续向量检索基础。
- MinIO：商品图片、视频和文件素材存储。
- Redis：缓存和异步任务基础能力。
- Docker Compose：本地和部署环境的基础服务编排。

## 项目目标

Jarvis 的长期目标不是做一个普通聊天机器人，而是做成公司内部真正可用的电商工作助手：

- 资料统一维护。
- 多端实时同步。
- 商品准确检索。
- 客服话术可复用。
- 导入和同步可追溯。
- 后期可连接电商平台和企业内部系统。

项目会按照“先把商品资料库做扎实，再让 AI 助手真正理解和使用这些资料”的路线逐步推进。
